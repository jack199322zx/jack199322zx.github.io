注意

这里要提醒的是，spring中异步任务的执行determineAsyncExecutor需要决定使用Executor，spring会优先从缓存中寻找executor，如果找不到则会根据@Async的value属性作为标识符进行判定，value的值默认为空串，如果有值，spring将会创建一个以此标识符命名的线程池，此时如果没有获取到是会抛出异常的；如果不设定value，spring会去获取配置的TaskExecutor，如果找不到配置的则会默认创建SimpleAsyncTaskExecutor。

事件驱动

ApplicationContext继承了ApplicationEventPublisher，可以调用publishEvent方法发布事件。
而事件的监听者可以通过实现ApplicationListener<? extends ApplicationEvent>重写onApplicationEvent来获取事件。

事件驱动原理

AbstractApplicationContext中定义了publishEvent方法。会获取事件广播器，委托其发布事件。
默认的事件广播器是SimpleApplicationEventMulticaster。

很多人有一个误区，认为这里getTaskExecutor就是从容器中获取配置的TaskExecutor，如果有就异步执行该任务。其实不是的，这里的getTaskExecutor返回的就是属性taskExecutor，默认为null。spring默认没有直接提供异步处理任务，@Async的本质还是使用代理类来异步执行任务，而如果想要不使用代理类，直接用自己定义的线程池，就需要自定义个事件广播器，显示调用setTaskExecutor配置线程池来完成异步任务处理。

最后

protected void invokeListener(ApplicationListener listener, ApplicationEvent event) {
    ErrorHandler errorHandler = getErrorHandler();
    if (errorHandler != null) {
        try {
            listener.onApplicationEvent(event);
        }
        catch (Throwable err) {
            errorHandler.handleError(err);
        }
    }
    else {
        try {
            listener.onApplicationEvent(event);
        }
        catch (ClassCastException ex) {
            // Possibly a lambda-defined listener which we could not resolve the generic event type for
            LogFactory.getLog(getClass()).debug("Non-matching event type for listener: " + listener, ex);
        }
    }
}

invokeListener将调用监听者的onApplicationEvent方法。
